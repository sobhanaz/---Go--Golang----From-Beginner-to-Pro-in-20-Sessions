# جلسه‌ی ۱۲ — همروندی ۱: گوروتین‌ها و کانال‌ها 🚦

**هدف (۱ ساعت):** با ویژگی‌ای که Go به‌خاطرش معروف است آشنا شوی. یک **گوروتین**
می‌گذارد تابعی را *به‌صورت همروند* (انگار هم‌زمان) اجرا کنی، تقریباً بدون هیچ
زحمتی، و **کانال‌ها** می‌گذارند گوروتین‌ها به‌صورت امن گفت‌وگو کنند.

> **مرور جلسه‌ی ۱۱:** خرابی‌ها را با مقادیر `error` مدیریت می‌کنی. همروندی چالش
> جدیدی می‌آورد — هماهنگی کاری که هم‌زمان اجرا می‌شود — و Go ابزارهای زیبایی دارد.

---

## ۱. همروندی در مقابل موازی‌سازی (۵ دقیقه)

- **همروندی (Concurrency)** = *سروکار داشتن* با چند چیز هم‌زمان (ساختار: کارهایی
  که مستقل اجرا می‌شوند و در زمان هم‌پوشان پیش می‌روند).
- **موازی‌سازی (Parallelism)** = *انجام دادن* چند کار در همان لحظه‌ی واقعی (چند
  هسته‌ی CPU که هم‌زمان کد اجرا می‌کنند).

Go بیان **همروندی** را آسان می‌کند؛ سپس رانتایم Go آن را برای رسیدن به موازی‌سازی
روی هسته‌ها زمان‌بندی می‌کند.

---

## ۲. گوروتین‌ها — همروندی با یک کلیدواژه (۱۵ دقیقه)

یک **گوروتین** تابعی است که مستقل و همروند اجرا می‌شود. با گذاشتن `go` قبل از
فراخوانی تابع، یکی شروع کن:

</div>

```go
go doWork()          // doWork را همروند اجرا می‌کند؛ main فوراً ادامه می‌دهد
go fmt.Println("hi") // روی هر فراخوانی تابع کار می‌کند
```

<div dir="rtl">

همین. گوروتین‌ها بسیار ارزان‌اند — می‌توانی *هزاران* تا اجرا کنی. اینها رشته‌ی
سیستم‌عامل نیستند؛ رانتایم Go گوروتین‌های زیاد را روی چند رشته مالتی‌پلکس می‌کند.

### نکته: main منتظر نمی‌ماند

وقتی `main` برمی‌گردد، برنامه خارج می‌شود — **حتی اگر گوروتین‌ها هنوز در حال اجرا
باشند.** این نسخه‌ی ساده اغلب هیچ چاپ نمی‌کند:

</div>

```go
func main() {
    go fmt.Println("from goroutine")
    // main فوراً تمام می‌شود، گوروتین ممکن است هرگز اجرا نشود
}
```

<div dir="rtl">

به راهی برای **منتظر ماندن** گوروتین‌ها نیاز داری. راه خام `time.Sleep` است (بد —
داری حدس می‌زنی). ابزارهای درست `sync.WaitGroup` (جلسه‌ی ۱۳) و **کانال‌ها** (بخش
بعد) هستند که می‌گذارند گوروتین‌ها *حرف* هم بزنند.

فایل [`examples/session12/goroutines/goroutines.go`](../examples/session12/goroutines/goroutines.go) را اجرا کن.

## ۳. کانال‌ها — ارتباط امن بین گوروتین‌ها (۲۵ دقیقه)

یک **کانال** یک لوله‌ی نوع‌دار است: یک گوروتین مقادیر را می‌فرستد، دیگری
دریافتشان می‌کند. کانال‌ها روشی هستند که گوروتین‌ها داده را **امن** به اشتراک
می‌گذارند — بدون قفل.

</div>

```go
ch := make(chan int)   // کانالی که int حمل می‌کند

ch <- 42               // ارسال 42 به کانال (فلش به داخل ch)
value := <-ch          // دریافت از کانال (فلش از ch به بیرون)
```

<div dir="rtl">

جهت فلش را به یاد بسپار: `ch <-` به *داخل* می‌فرستد، `<-ch` از *بیرون* دریافت می‌کند.

### کانال‌ها خودکار همگام‌سازی می‌کنند

به‌صورت پیش‌فرض کانال‌ها **بدون بافر**اند: ارسال تا وقتی گوروتین دیگری دریافت کند
مسدود می‌شود، و دریافت تا وقتی کسی بفرستد مسدود می‌شود. این مسدودسازی یک *ویژگی*
است — دو گوروتین را همگام می‌کند:

</div>

```go
func main() {
    ch := make(chan string)

    go func() {
        ch <- "result from goroutine"   // تا main دریافت کند مسدود می‌شود
    }()

    msg := <-ch   // تا گوروتین بفرستد مسدود می‌شود؛ این «انتظار» ماست!
    fmt.Println(msg)
}
```

<div dir="rtl">

اینجا دریافت `<-ch` باعث می‌شود `main` منتظر گوروتین بماند — بدون حدس `time.Sleep`.
شعار همروندی Go: **«با اشتراک حافظه گفت‌وگو نکن؛ با گفت‌وگو حافظه را به اشتراک بگذار.»**

### کانال‌های بافردار

یک کانال بافردار چند مقدار را قبل از مسدودشدن نگه می‌دارد:

</div>

```go
ch := make(chan int, 3)   // ظرفیت 3
ch <- 1
ch <- 2                   // تا بافر پر نشود مسدود نمی‌شوند
```

<div dir="rtl">

### بستن و range روی کانال

فرستنده می‌تواند کانال را `close` کند تا «دیگر مقداری نیست» را علامت بزند. گیرنده
می‌تواند روی کانال `range` بزند تا تا بسته‌شدنش بخواند:

</div>

```go
func produce(ch chan int) {
    for i := 1; i <= 5; i++ {
        ch <- i
    }
    close(ch)   // علامت: ارسال تمام شد
}

ch := make(chan int)
go produce(ch)
for n := range ch {   // 1..5 می‌خواند، بعد با بسته‌شدن ch متوقف می‌شود
    fmt.Println(n)
}
```

<div dir="rtl">

> 🔑 **قواعد کانال:**
> - فقط **فرستنده** باید کانال را `close` کند، هرگز گیرنده.
> - ارسال روی کانال بسته **panic** می‌کند. دریافت از کانال بسته فوراً مقدار صفر
>   برمی‌گرداند (با comma-ok `v, ok := <-ch` بسته‌شدن را تشخیص بده: `ok` وقتی بسته
>   و خالی شد `false` است).

فایل [`examples/session12/channels/channels.go`](../examples/session12/channels/channels.go) را اجرا کن.

## ۴. کانال‌های جهت‌دار (۱۰ دقیقه)

توابع می‌توانند اعلام کنند که فقط روی کانال **می‌فرستند** یا فقط **دریافت** می‌کنند.
این قصد را مستند و کامپایلر را قادر به گرفتن اشتباه می‌کند:

</div>

```go
func producer(out chan<- int) {   // chan<- : فقط ارسال
    out <- 1
}

func consumer(in <-chan int) {    // <-chan : فقط دریافت
    fmt.Println(<-in)
}
```

<div dir="rtl">

یک `chan int` ساده هر دو را می‌کند؛ `chan<-` فقط-ارسال؛ `<-chan` فقط-دریافت.
استفاده از نوع‌های جهت‌دار در امضای توابع نشانه‌ی Go تمیز است.

فایل [`examples/session12/directions/directions.go`](../examples/session12/directions/directions.go) را اجرا کن.

> ⚠️ **آشکارساز ریس (race detector) دوست توست.** وقتی گوروتین‌ها اشتباهاً به داده‌ی
> مشترک دست می‌زنند، یک *data race* می‌گیری (باگ ظریف و بد). برنامه‌های همروندت را
> با `go run -race ...` اجرا کن تا بگیریشان. در جلسه‌ی ۱۳ به آن تکیه می‌کنیم.

## 🎯 تمرین‌ها (قبل از جلسه‌ی ۱۳ انجام بده!)

فایل `examples/session12/practice/practice.go` را بساز:

۱. **سلام گوروتین:** گوروتینی که پیامی چاپ می‌کند راه بینداز و با یک کانال بدون بافر کاری کن `main` قبل از خروج منتظرش بماند.
۲. **جمع با کانال:** `func sum(nums []int, ch chan int)` که مجموع را به کانال می‌فرستد. یک اسلایس را نصف کن، هر نیمه را در گوروتین خودش جمع کن، بعد هر دو جمع جزئی را دریافت و جمع کن.
۳. **تولیدکننده/مصرف‌کننده:** تولیدکننده‌ای که ۱ تا ۱۰ را به کانال می‌فرستد و می‌بندد، و مصرف‌کننده‌ای (در `main`) که با `range` چاپشان می‌کند.
۴. **خط لوله‌ی مربع:** گوروتینی که اعداد را روی یک کانال دریافت و مربعشان را روی کانال دیگر می‌فرستد. ۱ تا ۵ را بده و مربع‌ها را چاپ کن.
۵. **نوع‌های جهت‌دار:** تمرین ۳ را با `chan<- int` برای پارامتر تولیدکننده و `<-chan int` برای مصرف‌کننده بازنویسی کن.

> 💡 هرکدام را با `go run -race examples/session12/practice/practice.go` اجرا کن تا عادت بررسی data race بسازی.
