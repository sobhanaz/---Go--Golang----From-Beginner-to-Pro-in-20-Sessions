# جلسه‌ی ۱۳ — همروندی ۲: الگوهای محصول واقعی ⚙️

**هدف (۱ ساعت):** از «می‌توانم یک گوروتین راه بیندازم» به «می‌توانم کد همروند درست
و واقعی بنویسم» برسی. `sync.WaitGroup` (انتظار برای چند گوروتین)، `sync.Mutex`
(محافظت از داده‌ی مشترک)، `select` (مدیریت چند کانال)، `context` (لغو و
وقفه‌ی زمانی)، و الگوی **استخر کارگر (worker pool)** که همه را به هم می‌بافد.

> **مرور جلسه‌ی ۱۲:** گوروتین‌ها همروند اجرا می‌شوند؛ کانال‌ها می‌گذارند گفت‌وگو و
> همگام شوند. حالا *چند* تا از آن‌ها را درست هماهنگ می‌کنیم.

---

## ۱. `sync.WaitGroup` — انتظار برای چند گوروتین (۱۵ دقیقه)

در جلسه‌ی ۱۲ از `time.Sleep` برای انتظار استفاده کردیم — یک حدس. ابزار واقعی
`sync.WaitGroup` است، شمارنده‌ای که می‌گذارد `main` تا اتمام همه‌ی گوروتین‌ها مسدود شود.

</div>

```go
var wg sync.WaitGroup

for i := 1; i <= 3; i++ {
    wg.Add(1)              // قبل از راه‌اندازی، شمارنده را زیاد کن
    go func(id int) {
        defer wg.Done()    // وقتی این گوروتین تمام شد، کم کن
        fmt.Println("worker", id)
    }(i)
}

wg.Wait()                  // تا رسیدن شمارنده به 0 مسدود می‌شود
fmt.Println("all done")
```

<div dir="rtl">

سه فراخوانی، همیشه به این شکل:
- `wg.Add(1)` — «الان یک گوروتین شروع می‌کنم.» *قبل* از `go` صدایش بزن.
- `defer wg.Done()` — «این گوروتین تمام شد.» اولین خط درون گوروتین.
- `wg.Wait()` — «اینجا تا اتمام همه مسدود شو.»

> ⚠️ **متغیر حلقه را به‌عنوان آرگومان پاس بده** (`go func(id int){...}(i)`) یا در Go
> نسخه‌ی 1.22+ به متغیرهای حلقه‌ی به‌ازای-تکرار تکیه کن. گرفتن یک متغیر حلقه‌ی
> مشترک با closure یک باگ کلاسیک است.

فایل [`examples/session13/waitgroup/waitgroup.go`](../examples/session13/waitgroup/waitgroup.go) را اجرا کن.

## ۲. `sync.Mutex` — محافظت از داده‌ی مشترک (۱۵ دقیقه)

وقتی چند گوروتین **همان** متغیر را می‌نویسند، یک **data race** می‌گیری — نتایج
خراب و غیرقابل‌پیش‌بینی. یک `sync.Mutex` (قفل انحصار متقابل) تضمین می‌کند هر بار
فقط یک گوروتین به داده دست بزند.

</div>

```go
type Counter struct {
    mu    sync.Mutex
    value int
}

func (c *Counter) Inc() {
    c.mu.Lock()         // قفل را بگیر
    defer c.mu.Unlock() // وقتی متد برمی‌گردد آزادش کن
    c.value++           // امن: هر بار فقط یک گوروتین اینجاست
}
```

<div dir="rtl">

جفت `Lock`/`defer Unlock` الگوی استاندارد است. هرچیز بینشان **بخش بحرانی** است —
محافظت‌شده از دسترسی هم‌زمان.

> 🔑 **دو راه برای اشتراک امن حالت در Go:**
> ۱. **کانال‌ها** — داده را *بین* گوروتین‌ها پاس بده (وقتی جا می‌افتد ترجیح داده می‌شود).
> ۲. **Mutex** — از داده‌ای که گوروتین‌ها واقعاً *مشترک* دارند محافظت کن (شمارنده، کش، مپ).
>
> هرکدام واضح‌تر است استفاده کن. با `go run -race` *اثبات* کن ریس وجود ندارد.

فایل [`examples/session13/mutex/mutex.go`](../examples/session13/mutex/mutex.go) را اجرا کن — با و بدون
`-race` امتحانش کن، و قفل را بردار تا آشکارساز ریس را ببینی فعال شود.

## ۳. `select` — انتظار روی چند کانال (۱۰ دقیقه)

`select` مثل یک `switch` برای کانال‌هاست: تا وقتی *یکی* از چند عملیات کانال آماده
شود منتظر می‌ماند، بعد آن case را اجرا می‌کند.

</div>

```go
select {
case msg := <-ch1:
    fmt.Println("from ch1:", msg)
case msg := <-ch2:
    fmt.Println("from ch2:", msg)
case <-time.After(time.Second):
    fmt.Println("timeout — nothing arrived in 1s")
default:
    fmt.Println("nothing ready right now (non-blocking)")
}
```

<div dir="rtl">

کاربردهای کلیدی:
- **اولین-آماده برنده** — هر کانالی که زودتر مقدار بدهد را مدیریت کن.
- **وقفه‌ی زمانی** — `time.After(d)` کانالی برمی‌گرداند که بعد از `d` فعال می‌شود؛
  با یک case در `select` جفتش کن تا یک انتظار را تایم‌اوت کنی.
- **بدون مسدودسازی** — یک case `default` اگر هیچ کانالی آماده نباشد فوراً اجرا می‌شود.

## ۴. `context` — لغو و مهلت (۱۰ دقیقه)

`context.Context` روشی است که Go **لغو (cancellation)**، **مهلت‌ها** و مقادیر
دامنه‌ی-درخواست را در گوروتین‌ها و فراخوانی‌های API منتشر می‌کند. در Go واقعی
همه‌جاست — هر هندلر HTTP و فراخوانی دیتابیس یک `context` می‌گیرد.

</div>

```go
ctx, cancel := context.WithTimeout(context.Background(), 2*time.Second)
defer cancel()   // همیشه cancel را صدا بزن تا منابع آزاد شوند

select {
case result := <-doWork(ctx):
    fmt.Println("got:", result)
case <-ctx.Done():
    fmt.Println("cancelled or timed out:", ctx.Err())
}
```

<div dir="rtl">

گوروتینی که به context احترام می‌گذارد `ctx.Done()` را بررسی می‌کند (کانالی که با
لغو یا انقضای context بسته می‌شود) و سریع متوقف می‌شود.

> 🔑 **الگو:** سطح بالا یک context می‌سازد (با مهلت یا تابع لغو)، آن را پایین پاس
> می‌دهد، و توابع داخلی `ctx.Done()` را برای خروج زودهنگام تماشا می‌کنند. همیشه
> `defer cancel()`.

فایل [`examples/session13/selectctx/selectctx.go`](../examples/session13/selectctx/selectctx.go) را اجرا کن.

## ۵. استخر کارگر — الگویی که واقعاً استفاده می‌کنی (۱۰ دقیقه)

یک **استخر کارگر (worker pool)** تعداد ثابتی گوروتین را اجرا می‌کند که وظایف را از
یک کانال «jobs» برمی‌دارند و نتایج را به یک کانال «results» می‌فرستند. همروندی را
محدود می‌کند (یک میلیون گوروتین نمی‌سازی) و اسب‌بارکش Go همروند است.

</div>

```go
func worker(id int, jobs <-chan int, results chan<- int, wg *sync.WaitGroup) {
    defer wg.Done()
    for j := range jobs {        // تا بسته‌شدن کانال، وظیفه بردار
        results <- j * j         // کار را انجام بده، نتیجه را بفرست
    }
}
```

<div dir="rtl">

`main` وظایف را می‌فرستد، کانال jobs را می‌بندد، منتظر کارگرها می‌ماند، بعد نتایج
را جمع می‌کند. این **همه‌چیز** را ترکیب می‌کند: گوروتین، کانال، نوع‌های جهت‌دار و
`WaitGroup`. این مثال را دقیق مطالعه کن — سوال رایج مصاحبه و بلوک سازنده‌ی سیستم‌های واقعی است.

فایل [`examples/session13/workerpool/workerpool.go`](../examples/session13/workerpool/workerpool.go) را اجرا کن.

## 🎯 تمرین‌ها (قبل از جلسه‌ی ۱۴ انجام بده!)

فایل `examples/session13/practice/practice.go` را بساز (هرکدام را با `-race` اجرا کن):

۱. **WaitGroup:** ۵ گوروتین راه بینداز که هرکدام ID خودش را چاپ کند. با `WaitGroup` کاری کن `main` منتظر همه بماند — بدون `time.Sleep`.
۲. **شمارنده‌ی امن:** `Counter` با `Mutex` بساز. ۱۰۰۰ گوروتین راه بینداز که هرکدام `Inc()` صدا بزند. مقدار نهایی را چاپ کن — باید دقیقاً ۱۰۰۰ باشد. با `-race` بدون ریس بودن را تأیید کن.
۳. **نمایش ریس:** همان شمارنده را *بدون* mutex بساز، با `-race` اجرا کن، و گزارش ریس (و مجموع غلط) را ببین. بعد درستش کن.
۴. **تایم‌اوت با select:** گوروتینی که زمان تصادفی می‌خوابد بعد روی کانال می‌فرستد. با `select` و `time.After(500ms)` یا نتیجه یا «timed out» را چاپ کن.
۵. **استخر کارگر:** استخری از ۳ کارگر بساز که مربع وظایف ۱ تا ۹ را حساب کنند. هر ۹ نتیجه را جمع و چاپ کن.

## ✅ چک‌لیست جلسه‌ی ۱۳

- [ ] می‌توانم با `WaitGroup` (Add/Done/Wait) منتظر چند گوروتین بمانم
- [ ] متغیرهای حلقه را به گوروتین پاس می‌دهم تا باگ گرفتن (capture) نشود
- [ ] می‌توانم داده‌ی مشترک را با `Mutex` (Lock / defer Unlock) محافظت کنم
- [ ] می‌فهمم کِی از کانال در مقابل mutex استفاده کنم
- [ ] می‌توانم `select` را برای اولین-آماده، تایم‌اوت و بدون-مسدودسازی استفاده کنم
- [ ] `context` را برای لغو/تایم‌اوت می‌فهمم و همیشه `defer cancel()` می‌کنم
- [ ] می‌توانم استخر کارگر با کانال jobs/results بسازم
- [ ] کد همروند را با `-race` اجرا می‌کنم
- [ ] هر ۵ تمرین را انجام دادم
