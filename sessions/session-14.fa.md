# جلسه‌ی ۱۴ — گشت‌وگذار در کتابخانه‌ی استاندارد 🧰

**هدف (۱ ساعت):** Go با یک **کتابخانه‌ی استاندارد** غنی و معروف عرضه می‌شود —
باتری‌ها همراهش هستند. پکیج‌هایی را که روزانه سراغشان می‌روی گشت می‌زنی: `strconv`
(رشته↔عدد)، `time` (تاریخ و بازه)، `sort` (مرتب‌سازی) و `os` (محیط برنامه). تسلط
خوب بر کتابخانه‌ی استاندارد بخش بزرگی از بهره‌وری در Go است.

> **مرور جلسه‌ی ۱۳:** مفاهیم هسته‌ی Go را تمام کردی. از اینجا بحث *ساختن* است — و
> کتابخانه‌ی استاندارد جعبه‌ابزار توست.

> 💡 **چطور هر پکیجی را کاوش کنی:** در ترمینال `go doc strconv` یا
> `go doc strconv.Atoi` را اجرا کن، یا [pkg.go.dev](https://pkg.go.dev) را مرور
> کن. با خواندن داکیومنت راحت شو — مهارتی روزمره برای توسعه‌دهنده‌ی Go است.

## ۱. `strconv` — رشته‌ها ↔ اعداد (۱۵ دقیقه)

از جلسه‌ی ۰۲ یادت هست که Go برایت رشته را به عدد تبدیل نمی‌کند، و `string(65)`
مقدار `"A"` می‌دهد نه `"65"`. پکیج `strconv` تبدیل‌های واقعی را انجام می‌دهد:

</div>

```go
n, err := strconv.Atoi("42")        // رشته -> int  («ASCII به عدد صحیح»)
s := strconv.Itoa(2026)             // int -> رشته  («عدد صحیح به ASCII»)
f, err := strconv.ParseFloat("3.14", 64) // رشته -> float64
b, err := strconv.ParseBool("true")      // رشته -> bool
```

<div dir="rtl">

> 🔑 **یک خطا برمی‌گردانند**، چون ورودی ممکن است عدد معتبری نباشد. همیشه بررسی‌اش
> کن — این همان الگوی `value, err` از جلسه‌ی ۰۵ در عمل است. تجزیه‌ی ورودی کاربر
> (فیلد فرم، پارامتر کوئری، تنظیمات) جایی است که در پروژه‌ی نهایی مدام استفاده‌اش
> می‌کنی.

فایل [`examples/session14/strconv/strconv.go`](../examples/session14/strconv/strconv.go) را اجرا کن.

## ۲. `time` — تاریخ، بازه و قالب‌بندی (۲۰ دقیقه)

</div>

```go
now := time.Now()                  // زمان فعلی
t := time.Date(2026, time.June, 13, 15, 4, 5, 0, time.UTC) // یک زمان مشخص
```

<div dir="rtl">

### بازه‌ها (Duration)

یک `time.Duration` زمان سپری‌شده را نشان می‌دهد. با ضرب یک واحد بسازشان:

</div>

```go
2 * time.Hour
500 * time.Millisecond
later := now.Add(48 * time.Hour)   // افزودن زمان
diff  := later.Sub(now)            // تفریق دو زمان -> یک Duration
fmt.Println(diff.Hours())          // 48
```

<div dir="rtl">

از بازه‌ها قبلاً در همروندی استفاده کردی (`time.After`، `time.Sleep`).

### قالب‌بندی و تجزیه — تاریخ مرجع منحصربه‌فرد Go

این همه را گیر می‌اندازد، پس یک بار یادش بگیر. به‌جای کدهایی مثل `%Y-%m-%d`، Go از
یک **تاریخ مرجع** به‌عنوان قالب استفاده می‌کند:

</div>

```
Mon Jan 2 15:04:05 MST 2006
```

<div dir="rtl">

(آن را ۱-۲-۳-۴-۵-۶ بخوان: ۱۵=۳ بعدازظهر، Jan=۱، دوم، ...، ۲۰۰۶.) قالب را *دقیقاً
به‌صورت همان تاریخ* می‌نویسی و Go مقدار تو را جایگزین می‌کند:

</div>

```go
t.Format("2006-01-02")          // -> "2026-06-13"
t.Format("Mon, 02 Jan 2006")    // -> "Sat, 13 Jun 2026"

parsed, err := time.Parse("2006-01-02", "2025-12-25") // رشته -> time
```

<div dir="rtl">

> 🔑 **`2006-01-02 15:04:05` را حفظ کن** — رایج‌ترین قالب و کلید همه‌ی قالب‌بندی
> تاریخ در Go است.

فایل [`examples/session14/timepkg/timepkg.go`](../examples/session14/timepkg/timepkg.go) را اجرا کن.

## ۳. `sort` — مرتب‌سازی داده (۱۵ دقیقه)

</div>

```go
sort.Ints(nums)       // مرتب‌سازی []int صعودی، سرجا
sort.Strings(words)   // مرتب‌سازی []string
sort.Float64s(fs)     // مرتب‌سازی []float64
```

<div dir="rtl">

### مرتب‌سازی استراکت (و هرچیز سفارشی) با `sort.Slice`

یک تابع «کمتر» (less) که ترتیب دلخواهت را توصیف می‌کند پاس بده:

</div>

```go
sort.Slice(people, func(i, j int) bool {
    return people[i].Age < people[j].Age   // صعودی بر اساس سن
})
```

<div dir="rtl">

برای نزولی `people[i].Age > people[j].Age` برگردان. تابع به این پاسخ می‌دهد: «آیا
عنصر i باید قبل از عنصر j بیاید؟»

### اصطلاح خروجی مرتب از مپ

مپ‌ها تصادفی پیمایش می‌شوند (جلسه‌ی ۰۷). برای چاپ مپ با ترتیب پایدار، کلیدها را در
یک اسلایس جمع، مرتب، بعد پیمایش کن:

</div>

```go
keys := make([]string, 0, len(m))
for k := range m {
    keys = append(keys, k)
}
sort.Strings(keys)
for _, k := range keys {
    fmt.Println(k, m[k])
}
```

<div dir="rtl">

فایل [`examples/session14/sortpkg/sortpkg.go`](../examples/session14/sortpkg/sortpkg.go) را اجرا کن.

## ۴. `os` — گفت‌وگو با سیستم‌عامل (۱۰ دقیقه)

</div>

```go
os.Args            // []string از آرگومان‌های خط فرمان (Args[0] = مسیر برنامه)
os.Getenv("HOME")  // خواندن یک متغیر محیطی ("" اگر تنظیم نشده)
os.LookupEnv("X")  // value, ok — «خالی» را از «تنظیم‌نشده» جدا می‌کند
os.Exit(1)         // همین حالا برنامه را با کد وضعیت تمام کن (0 = موفقیت)
```

<div dir="rtl">

> ⚠️ **`os.Exit` توابع defer‌شده را رد می‌کند!** اگر پاک‌سازی `defer` کرده باشی، با
> `os.Exit` اجرا نمی‌شود. بهتر است خطاها را تا `main` بالا برگردانی و آنجا خارج شوی.

متغیرهای محیطی روشی هستند که اپ‌های واقعی تنظیمات (آدرس دیتابیس، پورت، رمزها) را
می‌گیرند — آن‌ها را در لایه‌ی تنظیمات پروژه‌ی نهایی می‌خوانی.

آن را *با آرگومان* اجرا کن:

</div>

```bash
go run examples/session14/ospkg/ospkg.go hello world
```

<div dir="rtl">

> 📦 **جواهرات دیگر کتابخانه‌ی استاندارد** که ارزش دانستن دارند (با چندتایشان بعداً
> آشنا می‌شویم): `math`، `math/rand`، `bufio`، `regexp`، `encoding/json` (جلسه‌ی
> ۱۵)، `net/http` (جلسه‌ی ۱۷)، `testing` (جلسه‌ی ۱۶). کتابخانه‌ی استاندارد بزرگ و
> باکیفیت است — قبل از افزودن وابستگی شخص‌ثالث سراغش برو.

## 🎯 تمرین‌ها (قبل از جلسه‌ی ۱۵ انجام بده!)

فایل `examples/session14/practice/practice.go` را بساز:

۱. **تجزیه‌ی امن:** `func parseIntOrDefault(s string, def int) int` بنویس که از `strconv.Atoi` استفاده و در صورت شکست `def` را برگرداند. با `"100"` و `"oops"` تست کن.
۲. **سن از سال:** سال تولد را به‌صورت رشته بخوان، تبدیلش کن و با فرض سال جاری ۲۰۲۶ سن را چاپ کن. ورودی بد را مودبانه مدیریت کن.
۳. **قالب‌بندی شمارش معکوس:** با `time.Date(2027, 1, 1, ...)`، بازه را از یک «اکنونِ» ثابت حساب و تعداد روزهای مانده را چاپ کن.
۴. **مرتب‌سازی افراد:** یک `[]struct{Name string; Score int}` بساز و بر اساس امتیاز **نزولی** مرتب کن، بعد جدول رتبه‌بندی را چاپ کن.
۵. **مپ مرتب:** یک `map[string]int` از شمارش کلمه بساز و ورودی‌ها را به ترتیب الفبایی کلمه چاپ کن.
